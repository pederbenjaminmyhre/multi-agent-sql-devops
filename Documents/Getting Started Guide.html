Getting Started Guide

Prerequisites

  • Python 3.12+
  • An OpenAI API key (or Anthropic API key)
  • Docker Desktop (for container builds)
  • Git and GitHub CLI (gh)

Use Case 1: Run a Local Review

Review a SQL script locally without deploying anything.

Step 1 — Clone and install

git clone https://github.com/pederbenjaminmyhre/multi-agent-sql-devops.git
cd multi-agent-sql-devops
python -m venv .venv
source .venv/Scripts/activate    # Windows
# source .venv/bin/activate      # macOS/Linux
pip install -e ".[dev]"

Step 2 — Configure environment

cp .env.example .env
# Edit .env and set your API key:
#   LLM_PROVIDER=openai
#   LLM_API_KEY=sk-your-key-here
#   LLM_MODEL=gpt-4o

Step 3 — Start the server

uvicorn src.main:app --reload

The API is now running at http://localhost:8000. Visit /docs for the interactive Swagger UI.

Step 4 — Submit a review

curl -X POST http://localhost:8000/review \
  -H "Content-Type: application/json" \
  -d '{"sql": "SELECT * FROM Orders WHERE Status = '\''Shipped'\''", "max_iterations": 3}'

The response is a JSON report with findings, cleaned SQL, documentation, and a changelog.

Use Case 2: Automatic PR Reviews (GitHub Actions)

This is the primary use case — the system reviews SQL changes automatically when a developer opens a pull request.

How it works

  1. A developer creates a branch and adds or modifies a .sql file.
  2. They open a pull request against main.
  3. GitHub Actions detects the .sql file change and triggers the SQL Review Agent workflow.
  4. The workflow sends the SQL to the review API hosted on Azure.
  5. The API runs all four agents (Linter, Performance, Doc-Genie, Skeptic) with critique-revise looping.
  6. The workflow posts a formatted review comment on the PR with a findings table.
  7. If any finding has severity error, the workflow fails and blocks the merge.
Try it yourself Create a branch, add a .sql file with intentional flaws (e.g., SELECT *, missing SET NOCOUNT ON), and open a PR. The review comment will appear within 30-60 seconds.

Use Case 3: Interactive Testing via Swagger

The fastest way to experiment with the API.

  1. Open https://ca-sql-review.happybay-c8b24a7e.eastus.azurecontainerapps.io/docs in your browser.
  2. Expand the POST /review endpoint.
  3. Click Try it out.
  4. Paste your SQL into the sql field of the request body.
  5. Click Execute and review the response.

Use Case 5: Deploy to Azure

# Login to Azure
az login

# Deploy all infrastructure
bash infra/deploy.sh

# Or step by step:
az group create --name rg-sql-devops-portfolio --location eastus
az deployment group create --resource-group rg-sql-devops-portfolio \
  --template-file infra/main.bicep --parameters infra/parameters.json

# Build and push the container
az acr login --name acrsqldevops
docker build -t acrsqldevops.azurecr.io/sql-review-agent:latest .
docker push acrsqldevops.azurecr.io/sql-review-agent:latest
Important After deploying, you must set the LLM_API_KEY environment variable on the Container App. This can be done through the Azure Portal or via the REST API. Never commit API keys to source control.
